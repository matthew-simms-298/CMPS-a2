## Part A: Problem Definition and Requirements

### Problem Statement

Our system is a smart notification system that will send personalized notifications to users based on the following criteria. The notifications that will be sent to the user of this system will be based on their authentication status (is logged in?), their role in the system (admin, editor, viewer), subscriptionstatus (active or inactive), and the permissions that the user has set. The permissions are the list of allowed notfications for that specific user.

### Requirements

#### Functional Requirements

- Authenticate user
- Check user role
- Verify subscription type and status
- Retrieve user permissions
- Send personalized notifications based on the above criteria

#### Non-Functional Requirements

- System should handle multiple users simultaneously
- Notifications sent in a timely manner
